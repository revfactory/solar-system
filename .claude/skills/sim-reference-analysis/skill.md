# Simulation Reference Analysis — 기존 시뮬레이터 분석 스킬

기존 태양계 시뮬레이션 웹사이트/앱을 체계적으로 분석하여 구현 벤치마크를 제공한다.

## 분석 워크플로우

### 1단계: 시뮬레이터 목록 수집
1. WebSearch로 "best solar system simulation website", "solar system 3D simulator" 검색
2. 웹 기반, 데스크톱, 모바일 카테고리별 정리
3. GitHub에서 오픈소스 시뮬레이터 검색

### 2단계: 직접 방문 분석
1. 각 웹 시뮬레이터를 WebFetch로 방문
2. HTML/JS 소스에서 기술 스택 추정 (Three.js, Babylon 등)
3. 기능 체크리스트 작성

### 3단계: UX 패턴 추출
1. 네비게이션 (행성 간 이동 방식)
2. 정보 표시 (데이터 패널 레이아웃)
3. 시간 제어 (재생 속도 UI)
4. 카메라 제어 (줌, 회전, 팬)
5. 반응형/모바일 대응
6. 접근성 (키보드 네비게이션 등)

### 4단계: 기능 우선순위 도출
분석 결과에서 공통적으로 포함된 기능 = 필수
차별화된 기능 = 권장/선택

## 분석 대상 우선순위

**Tier 1 (반드시 분석):**
- NASA Eyes on the Solar System — 최고 정확도 벤치마크
- Solar System Scope — 비주얼 벤치마크
- GitHub 상위 오픈소스 (Three.js 기반) — 코드 벤치마크

**Tier 2 (가능하면 분석):**
- The Sky Live, Celestia Web
- Chrome Experiments: 100,000 Stars
- Universe Sandbox (기능/UX 참조)

## 품질 기준
- 실제 방문(WebFetch) 없이 리뷰만 읽는 것은 불충분하다
- 기술 스택은 소스/DevTools 기반으로 확인 가능한 것만 "확인됨", 나머지는 "추정"으로 구분
- 기능 분류(필수/권장/선택)의 근거를 명시한다
